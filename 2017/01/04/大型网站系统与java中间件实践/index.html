什么是分布式系统

各个组件分布在不同的网络计算机上，组件之间仅仅荣哟消息传递来通信并协调行动。

大型网站发展：

  1. 单机 ，应用和数据库放在一起。
  2. 单机+缓存，应用和数据库之间添加一层缓存。
  3. 应用和数据库拆分，缓存在应用层。
  4. 数据库读写分离
  5. 应用部署集群，
  6. 分布式缓存，集中缓存
  7. 搜索引擎(lucene,solr)，分布式文件系统。
  8. 应用垂直拆分(分压，解耦)，
  9. 数据库垂直拆分(分压，解耦)
  10. 服务化(解耦去重复)通信rpc
  11. 解耦消息中间件(吞吐量)
  12. 数据库水平拆分，分库分表。

分布式系统的意义在于：

  1. 升级单机处理能力的性价比越来越低
  2. 单机处理能力存在瓶颈
  3. 出于稳定性和可用性的考虑

分布式系统会遇到哪些问题？

  1. 全局时钟
  2. 全局sequence
  3. 全局事务
  4. 数据个维度查询
  5. 分布式锁
